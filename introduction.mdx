## Overview

U17g provides a centralized documentation space where you organize, manage, and share all your project documentation effortlessly. You create custom spaces for different projects, collaborate with teams in real-time, and ensure your docs stay up-to-date with version control and search capabilities. Whether you document APIs, user guides, or internal wikis, U17g simplifies the process with intuitive tools and integrations.

This platform scales with your needs, from small teams to enterprise projects, offering features like markdown editing, embeddable components, and analytics on doc usage.

## Why Choose U17g

You select U17g for its balance of simplicity and power. Unlike static site generators, U17g offers a live, collaborative environment without needing devops setup. It integrates seamlessly with tools like GitHub for version syncing and Slack for notifications.

Key benefits include reduced documentation silos, faster onboarding for new team members, and analytics to track which docs get the most views. Teams report `>50%` faster doc updates compared to traditional wikis.

## Quick Start

Get up and running in minutes with these steps.

<Steps>

<Step title="Create Account" icon="user-plus">

  Visit `https://app.u17g.com` and sign up with your email or GitHub account.

</Step>

<Step title="Set Up Space" icon="folder-plus">

  Create a new documentation space for your project. Name it and select a template.

  ```
  Example space config:
  {
    "name": "My Project Docs",
    "template": "api-reference"
  }
  ```

</Step>

<Step title="Add Your First Doc" icon="file-plus">

  Click "New Page" and start writing in markdown. Preview changes live.

  <CodeGroup tabs="Markdown,HTML">

  ```markdown
  # Welcome to My Docs

  This is your first page.
  ```

  ```html
  <h1>Welcome to My Docs</h1>
  <p>This is your first page.</p>
  ```

  </CodeGroup>

</Step>

<Step title="Publish and Share" icon="share-2">

  Hit publish to make it live. Generate a shareable link or embed code.

</Step>

</Steps>
