<Update label="2024-10-15" description="v1.2.0" tags={["feature", "bugfix"]}>

## New Features

- Added real-time collaboration support for shared documents, enabling multiple users to edit simultaneously.
- Introduced customizable themes with your brand color `#3B82F6` as the default accent.
- New API endpoints for bulk operations on projects.

## Improvements

- Enhanced search functionality with fuzzy matching and advanced filters.
- Optimized performance for large documentation spaces, reducing load times by 40%.

## Bug Fixes

- Fixed issue where embedded images failed to load in certain browsers.
- Resolved synchronization errors during offline edits.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "improvement"]}>

## New Features

- Integrated version history with diff viewer for tracking changes over time.
- Added export options to PDF, Markdown, and HTML formats.

## Improvements

- Streamlined the project organization interface with drag-and-drop folders.
- Better mobile responsiveness for documentation viewing.

## Bug Fixes

- Corrected permission errors when sharing links externally.
- Fixed keyboard navigation issues in the editor.

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["feature", "breaking"]}>

## New Features

- Basic authentication with API key support.

## Breaking Changes

## Initial Setup

- First-time user onboarding wizard.

</Update>

## How to Stay Updated

Follow these steps to ensure you never miss an update:

<Steps>
  <Step title="Watch on GitHub" icon="github">
    Visit the [U17g repository](https://github.com/example/u17g) and click Watch to get release notifications.
  </Step>
  <Step title="Enable Notifications" icon="bell">
    In your GitHub settings, select releases and discussions.
  </Step>
  <Step title="RSS Feed" icon="rss">
    Add `https://github.com/example/u17g/releases.atom` to your feed reader.
  </Step>
</Steps>

<Tabs>
  <Tab title="GitHub Releases" icon="github">

    Check the [releases page](https://github.com/example/u17g/releases) for full changelogs and download assets.

  </Tab>
  <Tab title="Migration Guide" icon="book-open">

    When upgrading, review the migration notes in each release:

    ```
    # Example migration for v1.2.0
    npm install u17g@1.2.0
    ```

  </Tab>
</Tabs>
